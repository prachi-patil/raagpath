Key architectural decision (v3.2): Next.js is built as a static export and bundled inside the Spring Boot WAR under /src/main/webapp/. Tomcat serves both frontend at / and backend at /api/* from a single Docker image. 01b —

Java 21 Feature Usage

🧵

Virtual Threads (Project Loom)

Every HTTP request runs on a lightweight virtual thread. No WebFlux reactive complexity needed — Spring Boot 3.3 enables this with one config flag. Handles thousands of concurrent practice sessions cheaply.

spring.threads.virtual.enabled=true NotificationService SessionService
📦

Records (Immutable DTOs)

All API request/response payloads modelled as Java Records — zero boilerplate, compile-time immutability. Serialized cleanly by Jackson.

record SessionRequest(...) All API DTOs XpAwardEvent
🔀

Sealed Classes + Pattern Matching

Gamification events modelled as a sealed hierarchy: XpEvent, BadgeEvent, StreakEvent. Switch expressions with exhaustive pattern matching replace verbose if-else chains in the event processor.

sealed interface GamificationEvent GamificationService
🔗

Structured Concurrency

RagaService fetches raga detail + reference audio URL + user completion status in parallel using StructuredTaskScope. Fails fast if any sub-task fails — cleaner than CompletableFuture chains.

StructuredTaskScope.ShutdownOnFailure RagaService
🔍

Pattern Matching for instanceof

Used throughout service layer to cleanly handle Optional unwrapping and polymorphic badge trigger conditions without verbose casting or null checks.

if (event instanceof XpEvent e) BadgeEngine
📝

Text Blocks

SQL queries in repositories written as clean multiline text blocks — no string concatenation, readable native queries for complex XP aggregation and leaderboard queries.

@Query("""...""") SessionRepository ProgressRepository
Sample — GamificationService.java
// Sealed event hierarchy
sealed interface GamificationEvent
    permits XpEvent, BadgeEvent, StreakEvent {}

record XpEvent(int amount, String reason) implements GamificationEvent {}
record BadgeEvent(String badgeId, String userId) implements GamificationEvent {}
record StreakEvent(int newStreak, boolean milestone) implements GamificationEvent {}

// Pattern matching switch in event processor
void process(GamificationEvent event) {
    switch (event) {
        case XpEvent(var amount, var reason)  -> awardXp(amount, reason);
        case BadgeEvent(var id, var uid)       -> unlockBadge(id, uid);
        case StreakEvent(var s, var milestone) -> updateStreak(s, milestone);
    }
}

AWS Infrastructure (Free Tier)

Free Tier Strategy: Every AWS service below is within the 12-month free tier. The entire MVP can run at $0/month during development and early launch. Watch RDS + EC2 hours — free tier gives 750 hrs/month each, which covers exactly 1 instance running 24/7.
🖥️

EC2 t2.micro

Runs the k3s Kubernetes cluster. Hosts Tomcat pod (Spring Boot WAR), Redis pod, and Nginx ingress. 750 hrs/month free — exactly 1 instance 24/7.

750 hrs/mo free 1 vCPU · 1 GB RAM k3s + Tomcat

RDS t3.micro (PostgreSQL 16)

Managed PostgreSQL outside the cluster — more reliable than in-pod DB for MVP data safety. 750 hrs/month free, 20 GB storage free. Multi-AZ disabled to stay in free tier.

750 hrs/mo free 20 GB storage Single-AZ

Stores raga audio samples, tanpura drone files, and Next.js static export. 5 GB free storage, 20K GET + 2K PUT requests/month free. Served globally via CloudFront.

Registry Decision — Why ECR Public over alternatives
✓ ECR Public (chosen)
Same AWS account. Zero pull rate limits. 50 GB free egress/mo. GitHub Actions pushes via OIDC — no stored AWS keys in secrets. k3s pulls with no credentials.
✗ Docker Hub (rejected)
Requires new account. GitHub Actions shared runners share IPs — free tier 200 pulls/6hr cap is hit quickly across CI runs. Third-party dependency.
✗ S3 Image Store (rejected)
K8s can't pull from S3 natively — needs a manual docker load script on EC2. Eats into 5 GB audio budget. No image tagging or version management.
✗ ECR Private (rejected)
Requires IAM imagePullSecret in k3s and credential rotation. Adds auth complexity for no MVP benefit. ECR Public gives same AWS residency, far simpler.
07b —

Docker + Kubernetes Design

Single image + ECR Public strategy: Next.js is statically exported (next export) during the CI build. The output is copied into src/main/webapp/ before the WAR is assembled — Tomcat serves frontend at / and API at /api/* from one container. The image is pushed to AWS ECR Public (public.ecr.aws/raagpath/app:latest) using GitHub Actions OIDC — no long-lived AWS credentials stored in CI. k3s pulls the image with no auth required since ECR Public repos are anonymously pullable. No imagePullSecret needed in K8s manifests.

Single Image: Next.js + Spring Boot WAR in Tomcat

Dockerfile — Single image (Next.js bundled inside Tomcat WAR)
# ── Stage 1: Build Next.js static export ──────────────────
FROM node:20-alpine AS frontend
WORKDIR /app/frontend
COPY frontend/package*.json ./
RUN npm ci
COPY frontend/ .
RUN npm run build # next export → /app/frontend/out/

# ── Stage 2: Build Spring Boot WAR ────────────────────────
FROM gradle:8-jdk21 AS backend
WORKDIR /app/backend
COPY backend/ .
# Copy Next.js static output into webapp root (served by Tomcat at /)
COPY --from=frontend /app/frontend/out/ src/main/webapp/
RUN gradle war --no-daemon # produces build/libs/raagpath.war

# ── Stage 3: Runtime — Tomcat 10 + JRE 21 ─────────────────
FROM tomcat:10.1-jre21-temurin AS runtime
RUN rm -rf /usr/local/tomcat/webapps/ROOT
COPY --from=backend /app/backend/build/libs/raagpath.war \
                     /usr/local/tomcat/webapps/ROOT.war
EXPOSE 8080
CMD ["catalina.sh", "run"]
How routing works in a single image: Tomcat serves / → Next.js index.html (static files from webapp root). /api/* → Spring MVC dispatcher servlet. /ws/* → Spring WebSocket STOMP endpoint. No Nginx needed — Tomcat handles everything behind Traefik ingress.
build.gradle.kts — WAR packaging
plugins {
    id("war")   // WAR for external Tomcat deployment
}

dependencies {
    // Mark embedded Tomcat as "provided" — external Tomcat container runs it
    providedRuntime("org.springframework.boot:spring-boot-starter-tomcat")
}

GitHub Actions CI/CD — ECR Public Push

.github/workflows/deploy.yml
name: Build & Deploy

on:
  push:
    branches: [main]

permissions:
  id-token: write   # required for OIDC — no long-lived AWS keys stored
  contents: read

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      # 1. Authenticate to AWS via OIDC (no stored secrets)
      - uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: arn:aws:iam::<account>:role/github-actions-ecr
          aws-region: us-east-1   # ECR Public is always us-east-1

      # 2. Login to ECR Public
      - run: |
          aws ecr-public get-login-password --region us-east-1 \
            | docker login --username AWS --password-stdin public.ecr.aws

      # 3. Build single image (Next.js + WAR + Tomcat)
      - run: |
          docker build -t public.ecr.aws/<alias>/raagpath:latest .
          docker build -t public.ecr.aws/<alias>/raagpath:${{ github.sha }} .

      # 4. Push both tags to ECR Public
      - run: |
          docker push public.ecr.aws/<alias>/raagpath:latest
          docker push public.ecr.aws/<alias>/raagpath:${{ github.sha }}

      # 5. SSH to EC2, tell k3s to pull the new image
      - run: |
          ssh -i ${{ secrets.EC2_KEY }} ec2-user@${{ secrets.EC2_HOST }} \
            "kubectl rollout restart deployment/raagpath"

Kubernetes Workloads on EC2

🚀

raagpath Deployment

1 replica of the single Tomcat pod — serves both Next.js frontend (/) and Spring API (/api/*). Image: public.ecr.aws/<alias>/raagpath:latest. No imagePullSecret needed — ECR Public is anonymously pullable. Liveness probe on /api/actuator/health.

replicas: 1 port: 8080 ECR Public · no auth
🔴

redis Deployment

Single Redis pod using official redis:7-alpine image. PersistentVolumeClaim for data persistence across restarts. ClusterIP service — not exposed externally.

redis:7-alpine PVC: 1Gi ClusterIP
🌐

Nginx Ingress Controller

k3s ships with Traefik by default — used as ingress. Routes /api/* to Tomcat service and /ws/* for WebSocket. TLS termination via Let's Encrypt cert-manager.

Traefik (k3s) TLS: Let's Encrypt /api + /ws routing
🔐

ConfigMaps + Secrets

DB credentials, JWT secret, AWS SES keys stored as K8s Secrets (base64). Non-sensitive config (Tomcat port, Spring profiles) in ConfigMap. Mounted as env vars.

K8s Secrets ConfigMap Env injection

⚠ Risk Flags — Where Days Slip
Pitch detection feel
Pitchy returns Hz accurately but the UX — debounce, smoothing, tolerance window — needs iteration. Don't skip this tuning on Day 3.
k3s + Traefik TLS
cert-manager on a fresh k3s install has sharp edges. Budget 1 hr for this specifically on Day 2. Do it before you need it on Day 5.
RDS connectivity
Security group rules between EC2 and RDS catch most people. Verify the Spring Boot → RDS connection on Day 1 before writing any app code.
Mobile mic access
Web Audio API on iOS Safari requires a user gesture before mic access. Test on mobile on Day 4, not Day 5.
